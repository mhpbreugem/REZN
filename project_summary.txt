## CORE CONCEPT: THE AGGREGATION SPACE

The aggregation space is the mathematical space in which equilibrium prices aggregate private signals. CARA demands are linear in log-odds → prices aggregate in log-odds space → matches the Bayesian sufficient statistic → full revelation. Non-CARA (log, CRRA) aggregates in a different space → mismatch → partial revelation. No noise traders needed.

## MODEL
- v ∈ {0,1}, prior P(v=1)=1/2
- K ≥ 3 groups, signals sₖ = v + εₖ, εₖ ~ N(0,1/τₖ)
- CRRA utility U(W) = W^(1-γ)/(1-γ)
- Supply z̄ = 0, NO noise traders
- Notation: Λ(z) = 1/(1+e⁻ᶻ) logistic, Λ⁻¹(μ) = ln(μ/(1-μ)) logit

## KEY DEMAND FORMULAS

### Private posterior
μₖ = Λ(τₖuₖ) where uₖ = sₖ − ½, and Λ⁻¹(μₖ) = τₖuₖ

### CARA demand
xₖ = [Λ⁻¹(μₖ) − Λ⁻¹(p)] / γₖ
Linear in log-odds gap. No wealth effects.

### Log demand  
xₖ = Wₖ(μₖ − p) / (p(1−p))
Linear in probability gap. Proportional to wealth.

### General CRRA demand (exact)
xₖ = Wₖ(R−1)/((1−p)+Rp), where R = exp((Λ⁻¹(μₖ)−Λ⁻¹(p))/γ)

### Market clearing
CARA: Λ⁻¹(p) = Σ wₖ·Λ⁻¹(μₖ), wₖ ∝ 1/γₖ → averages in log-odds
LOG: p = Σ wₖ·μₖ, wₖ ∝ Wₖ → averages in probability space

## MAIN RESULTS

### Prop 1: CARA = FR always
For ANY K≥1, ANY γₖ, τₖ, Wₖ: CARA equilibrium is fully revealing. Proved by showing the linear coefficient iteration a→(τ,τ,τ) for all heterogeneous γ configs (takes up to 500 iterations but always converges). Extends to entire exponential family.

### Prop 2: Log = PR
p = ΣΛ(τuₖ)/K is NOT a function of T* = Στuₖ by Jensen's inequality.

### Prop 3: Jensen Gap
Δp = −(τ³/48K)[U₃−U₁³/K²]+O(τ⁵)

### Prop 4: PR survives in full REE
Agent k learns Mₖ = Σⱼ≠ₖΛ(τuⱼ) — level sets are curved (Λ nonlinear) vs hyperplanes for CARA. Confirmed numerically: β_signal > 0 at the fixed point across all methods.

### Prop 5: Positive welfare
PR equilibrium has positive volume and E[U]. FR has zero trade.

### Prop 6: Value of info
V(τ)=0 under CARA; V(τ)>0 under non-CARA.

### Prop 8: Smooth transition
1−R² monotonically increasing as γ decreases from ∞. CARA is knife-edge.

## THE BB FIXED POINT PROBLEM

### Unknown: G³ prices P[i,j,l]

### For each triple (i,j,l) with price p = P[i,j,l]:
1. Agent 1 (signal i): extract contour {(j',l'): P[i,j',l']=p} from her G×G surface
2. Integrate f_v(u_j')·f_v(u_l') along contour → A_v(i,p)
3. Posterior: μ₁ = f₁(i)·A₁/(f₀(i)·A₀+f₁(i)·A₁)
4. Same for agents 2 and 3
5. Market clearing: Σ demand(μₖ, p) = 0

### Fixed point: P = P_new where P_new comes from step 5

### Key insight: posteriors use PRIOR signal densities f(u|v), NOT other agents' posteriors. All learning is encoded in P.

### Contour method (cleanest):
- For each agent at own-signal i, sweep u_A on grid, root-find u_B such that P_interp(i, u_A, u_B) = p
- Integrate density along the resulting curve
- No binning, no kernels, continuous precision

## CONVERGENCE ISSUES
- Picard iteration oscillates for low γ (nonlinear demands)
- Newton requires smooth F (kernel-based posteriors) but kernel introduces artificial PR
- BB discrete grid has snap-to-grid artifacts
- Contour method is cleanest but needs smooth price interpolation
- All methods agree qualitatively: PR survives at REE
